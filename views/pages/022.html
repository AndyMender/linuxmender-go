One of the major concerns that beginner Linux users have is whether they will still be able to run their favorite Windows programs, especially games, despite not running Windows itself. In fact, there are numerous distributions that address this issue as part of their missive. For instance, Zorin OS and Linux Mint advertise themselves as perfect not only for beginners, but also for former Windows users. Those two distributions I would surely recommend to Windows refugees. Apart from them, Robolinux offers a pre-configured virtual machine for Windows XP and Windows 7, so that Windows can run seamlessly alongside Linux.

However, that's not all! There is one majestic piece of software available to most Linux distributions – WINE. Contrary to popular belief, Wine Is NOT an Emulator (that's also what the acronym stands for, by the by). Windows emulators of course exist in the form of virtual machines. However, WINE works slightly differently. It is a compatibility layer which directly translates Linux system calls to their closest Windows-specific equivalents. Due to that and because of the way the code is written, the resource penalty of running Windows applications through WINE is minute. Hell, some run even smoother than natively on Windows.

Supposedly, modern 64-bit Windows versions also use a compatibility layer for 32-bit applications. Although, I no longer own a Windows computer to verify this. Anyway, I decided to give some old gaming classics a go. As a case study I have chosen Age of Empires 2: The Age of Kings.

It's worth mentioning that the main website of the WINE project, WineHQ, hosts a vast database of reports on Windows applications. Some had already been tested and are known to work splendidly. Some, ranked Garbage don't and most likely never will. Such is the binary-only nature of commercial software, though. Age of Empires 2 was a good choice as it had been ranked Gold (playable, with only minor issues). Admittedly, the issues I encountered were a little more than minor:

  • 1. CD-ROM not recognized.

    This might come as a surprise, but Linux distributions are sometimes incompletely configured and they do not automagically recognize and / or mount the game CD-ROM. There is a quick fix to that, though. In our filesystem tab file /etc/fstab we define the following line:
    /dev/sr0 /media/cdrom iso9660 ro,exec,noauto 0 0
    This translates to mount block device /dev/sr0 (the usual CD-ROM path) of filesystem type ISO9660 locally as /media/cdrom, read-only (ro) and for execution (exec). The noauto option guarantees that the mount procedure will not be executed automatically. The reason I am actually listing all of that is because I had to do quite a bit of research to find a working solution.

  • 2. Game crashes at screen resolution 1024×768.

    This happens only when in full-screen mode. I believe the reason is a collision between WINE and the window manager. Alas, I have a small, 13” screen with a 1366×768 screen resolution. Not exactly suited for gaming.

On a positive note, these were the only issues I encountered when trying to run Age of Empires 2 through WINE. No graphical glitches (a common problem with WINE from what I read). Sound worked well, too. As I remember, I had had many more issues with Age of Empires 2 back on Windows 7.
