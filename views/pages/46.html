The penguin front is growing strong, gaining popularity with each and every passing day. The very same can be said about the ancestral Unices (Unixes?) - the BSDs. Incidentally, the BSDs have improved greatly over the years. Many of the complaints against them are no longer valid. Sadly, these outdated complaints often deter prospective users from even trying modern Unices. Hence, I decided to do the BSDs, and more specifically FreeBSD, some justice and give it a fair go.

To begin with, the legendary ZFS file system is as strong as ever, with new features and fixes being constantly introduced. Frankly, in terms of safe (and redundant) storage of data, it is still unparalleled. I believe ZFS alone is a strong argument for using FreeBSD to run enterprise servers. dtrace, a tool for system performance auditing, is extensively used by companies as well.

A new feature largely expanded in FreeBSD 10 is bhyve. For many years it was painfully obvious that FreeBSD lacks fully native virtualization capabilities. The legendary jails were of course in common use, but constituted only a para-virtualization feature (FreeBSD guests on a FreeBSD host). Currently, bhyve supports guest sessions for BSDs (OpenBSD and FreeBSD), Linux and even Windows 10!

In terms of common software, the FreeBSD Ports Collection offers pretty much everything a Linux distribution like Debian provides (over 24,000 packages in total). Honestly though, only approximately 10% of that is in everyday use. Here's a list of common tasks/features and the software which covers them:

  • Internet Browsing

    Firefox, Chromium (crashes often, but works), old version of Opera

  • Music and Video

    Parole, mplayer, mpv, VLC, Xine, etc.

  • Image Viewing and Processing

    GImageView, Inkscape, Feh, GIMP, etc.

  • Office Software

    LibreOffice, OpenOffice, AbiWord, Gnumeric

  • 3D Rendering and Animation

    Blender, CAD suites and libraries, K-3D (libre 3D modeling)

  • Programming Environments and Editors Geany, Eclipse, IDEA, Emacs, etc. ( + almost all programming languages)
  • Gaming

    Many native ports of open-source games and Linux packages with compatibility layers

Unfortunately, there are some areas in which FreeBSD falls short for now:

  • Non-native File System Support

    Full NTFS (Windows) support, but lacking support for Linux file systems (only ext2 and ext3 are supported in the kernel)

  • Graphics Support

    ATI/AMD cards only via the open-source radeon driver (no Catalyst), nVidia cards almost only through the closed-source driver.

  • Wireless Support

    Only selected wireless chips from Intel, Atheros, Realtek and a couple of other companies have kernel-level support, hence mobile hardware has to be chosen carefully. Intel for everything is a safe bet, though.

  • Closed-source Software

    Third-party software distributed in binary form only is available for Linux, but not for any of the BSDs The Linux compatibility layer may help, but is not a sure solution.

  • Linux Compatibility Layer

    It's there, but the Linux libraries are derived from older CentOS releases.

As is clear from the features I listed, FreeBSD is more of a server operating system. Still, I would strongly recommend FreeBSD as a Linux or Windows alternative. It is a rock-solid OS with sane and predictable system management utilities. Frankly, one can actually get bored by how well it works!
